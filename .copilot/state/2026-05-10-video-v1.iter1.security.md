# Security review — Video Intelligence V1 plan

> Reviewer: Security agent · Date: 2026-05-10 · Iteration: 1
> Plan reviewed: [.copilot/state/2026-05-10-video-v1.plan.md](2026-05-10-video-v1.plan.md)

## Scope of this review

This is a plan-time review (per planner heuristic: "Flag anything touching
auth, secrets, PII, or dependencies — the Security agent must review the plan
before build starts"). Code review will follow on the PR.

V1 is pure scaffolding. It introduces no upload, no streaming, no AI, no
worker, no third-party packages, and no public surface (controller is
404-gated by default). The lens for this pass is therefore: **does the plan
set up a foundation that won't paint us into a corner on the security
non-negotiables when later slices land?**

## Standing rules carried forward to all future video slices

1. **Every** video endpoint calls `TeamScope.RequireOwnedTeam` (or a stricter
   variant) before any DB read/write. Cross-team isolation tests required.
2. **Signed URLs** must HMAC `(assetId, expiresAt, viewerUserId)` with a
   server-side secret pulled from config (never literal); TTL ≤ 15 minutes.
3. **Uploads** (V2+): MIME allow-list checked **after** ffprobe verification,
   not from client-supplied `Content-Type`. Per-team byte quota enforced
   server-side. Filename never trusted — store under
   `teams/{teamId}/{assetId}/source.<ext>` with extension derived from probe.
4. **Worker DB role** (V3+): separate Postgres role with `INSERT/UPDATE` only
   on `video.*` tables — no `DELETE`, no access to non-video tables.
5. **AI prompts** (V6+): built only by `IInsightService`; raw welfare-flagged
   timeline events **never** included. Test must assert this.
6. **Logs**: no transcript text, no voice-note transcripts, no welfare body
   text in logs. Use `WelfareDestructuringPolicy` pattern.
7. **New dependencies**: any addition (ffmpeg wrapper, Whisper client, S3
   SDK, etc.) requires a fresh Security review before merge.

## Decision

**PASS — backend may proceed with tasks 1–6** subject to the three small
adjustments in findings #3, #4, #5 (low-severity, planner can absorb them
without re-planning):

- Add a second cross-team isolation test on `VideoTimelineEvent` (AC7
  expansion).
- `IObjectStore.GetSignedReadUrlAsync` signature includes `viewerUserId` and
  TTL.
- `VideoJob` includes `TeamId` and `TraceParent`.

Backend should land all three in the same PR. Security will re-review on the
PR diff.
