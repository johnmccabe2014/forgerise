## Surface under review

| Area | Change |
| --- | --- |
| api / data | Entities `Session`, `AttendanceRecord`, `SessionPlan` (+ enums `SessionType`, `AttendanceStatus`). Indexes `(TeamId, ScheduledAt)`, `(SessionId, PlayerId)` unique, `(TeamId, GeneratedAt)`. Soft-delete on `Session`. EF migration `Sessions`. |

## Threats considered

| ID | Threat | Mitigation |
|---|---|---|
| S1 | Plan generator leaks raw welfare into stored plan | Generator is fed `(PlayerId, SafeCategory)` only. `SessionPlanContext` has no field for raw welfare. Endpoint test `Generate_after_checkin_includes_readiness_snapshot_without_raw` writes a high-risk check-in with sentinel raw text and asserts the response body never contains it. |
| S2 | Cross-team data exposure via plan or attendance | All endpoints behind `TeamScope.RequireOwnedTeam`. Attendance bulk upsert verifies every `playerId` belongs to the team or rejects 400. Test `Player_outside_team_is_rejected` covers this. |
| S3 | Future LLM provider exfiltrates raw via prompt | `ISessionPlanGenerator` only ever receives the safe context. The interface itself is the redaction boundary; reviewers can grep for any code path that wraps it with raw data. |
| S4 | Coach review notes leak into welfare logs | Review notes are coach-authored about coaching, not athlete welfare; not in `RawWelfareFields.Names`. We still avoid logging the body — `sessions.reviewed` log carries id + team only. |
| S5 | Attendance race conditions create duplicate rows | Unique index `(SessionId, PlayerId)` on `AttendanceRecord` plus upsert-by-existing-dictionary inside a single SaveChanges. |
| S6 | Plan generation request loops via `BasedOnSessionId` from another team | Controller verifies the basis session belongs to the target team. |
| S7 | Free-form review notes XSS surface | Stored as text; controllers do no rendering. UI must escape on render — note for frontend phase. |
| S8 | Unbounded attendance bulk size | `MinLength(1)` only; no max length set. Practical roster sizes are small (<60). Tracked. |
